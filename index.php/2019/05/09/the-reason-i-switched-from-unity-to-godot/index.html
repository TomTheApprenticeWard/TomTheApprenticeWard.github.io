The reason I switched from Unity to Godot.

The reason I switched isn’t that godot is better but because it runs much better on my computer. I have an older laptop and unity just runs poorly on it. Unity is slow on my laptop and takes up a large portion of the RAM. I can barely have a browser open at the same time. In addition, Unity is optimized for screens that are much larger or multi monitor setups. Even godot is a bit cramped on my laptop monitor, but it is considerably better.

I will miss using a statically typed language like C#, but it is not worth dealing with the poor performance of Unity. Of course I could buy a better computer, but I am a hobbyist game developer so it doesn’t make financial sense at this time.

Aside from its poor performance I think I generally prefer unity to godot (although I just started with godot so I can’t say for sure). But Godot does have a number of advantages: open source, smaller size on disk, smaller game projects on disk, optimized for Linux (I think), and of course much better performance including startup speed. Godot seems promising.
